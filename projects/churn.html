PROJECT

Telecom Customer Churn Prediction

A Hybrid XGBoost + SMOTE-ENN Framework: an end-to-end solution combining preprocessing, resampling, and ensemble learning to optimize customer churn prediction.

XGBoost SMOTE-ENN Imbalanced Learning Feature Selection Telecom Analytics

Background

  • Customers are a critical asset — in the telecom industry, customers are the strategic, core asset that determines a company's success.
  • Churn causes major losses — customers leaving causes revenue loss and directly hurts profit, making an early-prediction strategy essential.
  • ML has become the go-to tool — machine learning has become an effective tool for analyzing behavior and predicting customer churn.
5–6x the cost to acquire a new customer vs. retaining an existing one 68% of customers leave because they feel the company doesn't care about them

Reducing churn by 5% can increase profit by 25–95% depending on the industry (Bain & Company research). Early churn prediction gives a business a "golden window" to take effective retention action before a customer decides to leave.

Research Gap

  • Data preprocessing — many studies focus on the ML algorithm but pay too little attention to data preprocessing, a factor that significantly affects model performance.
  • Imbalanced data — few studies thoroughly address class imbalance in churn prediction, leading to inaccurate results for the minority class.
  • Combining multiple techniques — current research rarely focuses on hybrid approaches to optimize churn prediction performance.
  • XGBoost + hybrid resampling — few studies combine XGBoost with a hybrid resampling technique like SMOTE-ENN to address class imbalance.
  • An integrated framework — no existing framework integrates all three factors — data preprocessing, ensemble learning, and resampling — into one complete solution.

Research Objectives

  • Build a hybrid framework combining preprocessing + SMOTE-ENN + XGBoost to improve churn prediction accuracy and reduce misclassification.
  • Compare model performance before and after class balancing with hybrid resampling.
  • Analyze the trade-off between processing speed and prediction accuracy.
  • Use XGBoost to identify the key features driving customer churn decisions in the telecom industry.
  • Compare the proposed framework's performance against prior studies on the same datasets.

Hybrid Framework — 3 Layers

LayerDetails
1. Data Layer
Preprocessing & data preparation		 Data cleaning → EDA → Feature engineering → Feature selection (Univariate) → Scaling (StandardScaler)
2. Algorithm Layer
Handling imbalance & model building		 SMOTE-ENN hybrid resampling → Stratified k-fold (k=10) → XGBoost as the main model → Hyperparameter selection
3. Evaluation Layer
Model performance evaluation		 Accuracy, Precision, Recall, F1-score, ROC-AUC

SMOTE generates synthetic samples for the minority class, ENN removes noisy and mislabeled samples — the hybrid combines the strengths of both. Data is split train:test = 70:30, using stratified k-fold (k=10) cross-validation. XGBoost was chosen as the main model because it handles imbalanced data very well, works well with tabular data containing non-linear features, and provides clear model interpretability through feature importance.

Experimental Setup

The study runs 2 experiment groups to evaluate the effectiveness of the resampling technique:

  • Experiment (a): XGBoost + SMOTE-ENN + StandardScaler
  • Experiment (b): XGBoost + StandardScaler (no resampling)

XGBoost parameters: max_depth=5, gamma=1, n_estimators=100, n_jobs=56.

Future Work

Time-Based Churn Prediction — Extending the Model to Time Series

  • The current model only considers static data at a single point in time, whereas customer behavior changes over time.
  • Extend to time-series or sequence modeling — promising technologies: survival analysis, LSTM, Transformer.
  • Predict churn risk at each point in time, better reflecting the customer's true behavioral cycle.

Realtime Monitoring & Early Warning

  • A real-time monitoring system that automatically ingests newly generated data.
  • Recompute churn probability on a short cycle (hourly/daily).
  • Detect early warning signals, raise timely risk alerts, and trigger a retention campaign as soon as churn indicators appear.
